# Evidence Roadmap for Publication

This roadmap defines the evidence needed to move `ml-lab` from a science-abstract ML system into a publishable research contribution. It separates engineering evidence, simulation evidence, methodological evidence, and empirical learning evidence.

## Publication thesis

`ml-lab` tests whether adaptive machine-learning scaffolds can improve durable learning, transfer, metacognitive calibration, and reduced dependency compared with static scaffolding, unguided LLM assistance, and no-AI control conditions.

The publishable contribution is strongest if the paper is framed as a reproducible system and research protocol first, with synthetic experiments used to validate workflow behavior before human-subject learning claims are made.

## Evidence ladder

| Evidence level | Question answered | Acceptable evidence | Publication claim supported |
| --- | --- | --- | --- |
| Conceptual evidence | Is the research question grounded in learning science and ML system design? | Verified literature matrix, construct definitions, research design document. | The study is theoretically motivated and methodologically coherent. |
| Engineering evidence | Does the system implement the claimed abstractions? | Tests for learner state, scaffold policies, experiment arms, event logs, and evaluation utilities. | The repository is a reproducible ML research scaffold. |
| Simulation evidence | Does the synthetic workflow behave as expected under controlled assumptions? | Fixed-seed synthetic runs, event logs, summaries, pairwise comparisons, sensitivity checks. | The workflow can execute and audit scaffold-policy comparisons. |
| Methodological evidence | Are outcomes, comparisons, and interpretation boundaries specified before analysis? | Pre-analysis plan, data dictionary, exclusion rules, outcome definitions, reproducibility notes. | The study design is transparent and publication-ready. |
| Empirical evidence | Do real learners benefit from adaptive scaffolding? | Human-subject data, preregistered analysis, retention/transfer outcomes, robustness checks. | Adaptive scaffolding improves learning outcomes under specified conditions. |

## Manuscript-ready claim boundaries

### Claims currently supportable

The current repository can support claims such as:

- `ml-lab` implements a modular scaffold-policy research architecture.
- The system can simulate four experimental arms from the research design.
- The workflow produces event logs, arm summaries, and pairwise comparisons.
- The documentation separates synthetic workflow validation from empirical learning claims.

### Claims not yet supportable

The current repository should not yet claim:

- adaptive scaffolding improves real learner outcomes;
- unguided LLM assistance causes dependency in real learners;
- the current synthetic learner model approximates a validated human learning process;
- observed synthetic effect sizes estimate real-world educational effects.

## Evidence milestones

## Milestone 1: Reproducible system paper baseline

Goal: make the repository publishable as a system/protocol contribution.

Required artifacts:

- Complete README and documentation spine.
- Literature matrix with verified APA entries and source categories.
- Pre-analysis plan template.
- Synthetic workflow documentation.
- Unit tests for core objects, synthetic runner, summaries, and comparisons.
- One reproducible synthetic run with generated outputs stored as a release artifact or clearly regenerated by command.

Possible paper claim:

> We present a reproducible ML research scaffold for comparing adaptive learning-support policies under explicit learning-science outcome definitions.

## Milestone 2: Synthetic methods paper

Goal: demonstrate that the abstract machine can test policy behavior under controlled learner assumptions.

Required artifacts:

- Configurable synthetic learner parameters.
- Sensitivity analysis across learner skill, confidence bias, support intensity, and task count.
- Transfer and delayed-learning placeholder tasks.
- Comparison report showing how policy behavior changes under assumptions.
- Clear statement that synthetic results validate workflow behavior rather than human learning effectiveness.

Possible paper claim:

> Synthetic learner experiments show how scaffold-policy assumptions propagate through event logs, calibration metrics, support intensity, and pairwise arm comparisons.

## Milestone 3: Pilot empirical protocol

Goal: prepare for a small human-subject pilot.

Required artifacts:

- Task bank with item metadata.
- Human-subject event schema.
- Consent-aware logging and de-identification plan.
- Pilot recruitment and assignment protocol.
- Pilot analysis script using learner-level and event-level outcomes.
- Missing-data and exclusion rules.

Possible paper claim:

> The protocol is ready for pilot testing of adaptive scaffold policies against static and unguided AI support conditions.

## Milestone 4: Pilot learning evidence

Goal: collect initial real-learner evidence.

Required artifacts:

- Pilot sample with randomized arm assignment.
- Immediate, delayed, and transfer assessments.
- Confidence ratings paired with correctness.
- Logged scaffold actions or AI-use traces.
- Learner-level summaries and repeated-measures models.
- Robustness checks excluding incomplete cases.

Possible paper claim:

> Pilot evidence estimates the feasibility and directional learning effects of adaptive scaffolding relative to static scaffolding, unguided LLM assistance, and no-AI control conditions.

## Milestone 5: Full empirical study

Goal: support strong learning-effect claims.

Required artifacts:

- Adequately powered sample.
- Preregistered analysis plan.
- Validated outcome measures.
- Transfer and retention assessment.
- Model-based treatment-effect estimates.
- Subgroup analysis by prior skill and calibration profile.
- Replication package.

Possible paper claim:

> Adaptive ML-driven scaffolding improves specified learning outcomes under defined learner, task, and support conditions.

## Target publication shapes

### System/protocol paper

Best near-term target. Emphasizes architecture, reproducibility, research design, and the bridge between learning-science constructs and ML objects.

Primary evidence needed:

- working package;
- tests;
- docs spine;
- synthetic demonstration;
- verified literature matrix;
- clear interpretation boundaries.

### Simulation methods paper

Medium-term target. Emphasizes synthetic learner modeling and scaffold-policy comparison under controlled assumptions.

Primary evidence needed:

- configurable simulation;
- sensitivity analysis;
- comparison plots or tables;
- policy behavior diagnostics;
- limitations of synthetic learner assumptions.

### Empirical education/AI paper

Longer-term target. Requires real learner data.

Primary evidence needed:

- task bank;
- human-subject protocol;
- randomized assignment;
- transfer and delayed outcomes;
- preregistered or locked analysis plan;
- robust reporting.

## Interpretation rule

Every result should be labeled as one of three types:

- engineering validation;
- synthetic simulation evidence;
- empirical learning evidence.

Do not let claims migrate upward on the evidence ladder without the required artifacts.
