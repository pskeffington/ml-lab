# Publishable Topic: Best Practices in Machine-Learning Scaffolding

## Working Title

**Best Practices for Adaptive Machine-Learning Scaffolding: Timing, Fading, Calibration, and Transfer in Human-AI Learning Systems**

## Core Claim

The central publishable opportunity is not to show that AI assistance can improve short-term task completion. The stronger contribution is to define and test best practices for machine-learning scaffolding that improve durable learning while reducing overreliance on AI support.

## Research Gap

Current AI-supported learning systems often provide assistance that is either unrestricted, static, or insufficiently tied to learner evidence. This creates a risk that learners complete tasks successfully while developing weak transfer, poor metacognitive calibration, and high dependency on hints or generated answers.

A publishable study should therefore evaluate how scaffolding should be delivered, not merely whether AI should be available.

## Proposed Research Question

Which scaffolding practices in ML-supported learning environments best improve transfer, metacognitive calibration, and independent problem solving while minimizing AI overreliance?

## Best-Practice Dimensions

### 1. Evidence-Based Scaffolding

Scaffolds should be selected from observable learner evidence: attempts, errors, confidence, time on task, prior hints, and concept history. The system should not provide the same support to every learner.

### 2. Timed Access

AI help should be available at pedagogically meaningful moments rather than continuously. Timing itself can function as a scaffold by forcing productive struggle before intervention.

### 3. Gradual Fading

Scaffold intensity should decrease as learner competence improves. Fading is central to preventing dependency and encouraging transfer.

### 4. Metacognitive Prompting

The system should ask learners to predict, justify, reflect, or compare alternatives before receiving direct answers. The goal is to preserve learner agency and improve calibration.

### 5. Hint Laddering

Support should progress from low-information prompts to more explicit hints only when needed. A useful ladder is:

1. Orientation prompt
2. Concept reminder
3. Diagnostic question
4. Worked micro-example
5. Partial solution
6. Full explanation after attempt

### 6. Transfer-First Evaluation

Evaluation should prioritize delayed post-tests, transfer tasks, and independent performance, not only immediate task completion.

### 7. Overreliance Measurement

The experiment should directly measure hint dependency, premature help-seeking, copy-forward behavior, and reduced independent success after scaffolds are removed.

## Proposed Experimental Conditions

1. No-AI control
2. Unrestricted LLM assistance
3. Static scaffold prompts
4. Adaptive ML scaffold policy
5. Adaptive timing-and-fading policy

## Primary Outcomes

- Delayed learning gain
- Transfer task performance
- Metacognitive calibration
- Hint dependency
- Productive struggle duration
- Independent post-scaffold performance

## Initial Contribution

This project can contribute a reusable best-practices framework, an experimental protocol, and an open-source scaffold policy simulator. The paper should position itself as both a methodological contribution and an empirical study of human-AI learning design.

## Practical Thesis

The best ML scaffolding systems should not maximize assistance. They should maximize appropriately timed independence.
